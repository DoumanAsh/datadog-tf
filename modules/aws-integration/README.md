# aws-integration

This module performs simple setup of AWS cloud integration within datadog to allow it to collect and monitor resources of the AWS cloud instance.

This requires to create policy with broad set of permissions to allow datadog's its data collection.
You can find details [here](https://docs.datadoghq.com/integrations/amazon-web-services/#aws-iam-permissions)

## Parameters

| Parameter               | Description |
|-------------------------|-------------|
| `aws_account_id`        | Your AWS account identifier |
| `environment`           | Descriptive name for your cloud which gets attached as tag `env`  |
| `datadog_api_key`       | Datadog API Key to use |
| `datadog_app_key`       | Datadog APP Key with at least following permissions: `aws_configuration_edit`, `aws_configuration_read` and `aws_configurations_manage` |
| `datadog_aws_account_id`| Datadog's AWS account who will assume role generated by this module to collect data. Defaults to US's account. [Reference](https://docs.datadoghq.com/integrations/guide/aws-manual-setup/?tab=roledelegation&site=us) |
| `datadog_api_url`       | Datadog's API site. Defaults to US. [Reference](https://docs.datadoghq.com/getting_started/site/) |
